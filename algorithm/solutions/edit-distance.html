Edit Distance

June 09, 2019

Link: Minimum Path Sum

Problem description

Given a m x n grid filled with non-negative numbers, find a path from top left to bottom right which minimizes the sum of all numbers along its path.

Note: You can only move either down or right at any point in time.

Example:

Input:
[
  [1,3,1],
  [1,5,1],
  [4,2,1]
]
Output: 7
Explanation: Because the path 1→3→1→1→1 minimizes the sum.

Method

Basic dynamic programming problem.

The state transition formula is as following:

(i,j) = min((i-1, j), (i, j-1)) + grid(i,j)

Note that first column and first row should be filled first, as the shortest path to them is directly move from start point. There is no negative cell in the grid.

Code

/**
 * Dynamic programming with 2D table.
 * (i,j) = min((i-1, j), (i, j-1)) + grid(i,j)
 * First row and column can be filled at beginning since only one down or right move can be made each time.
 *
 * @param grid input 2D array grid
 * @return min path length from top left to bottom right
 */
public int minPathSum(int[][] grid) {

    /* Corner case */
    if (grid.length == 0) {
        return 0;
    }

    for (int i = 1; i < grid.length; i++) {
        grid[i][0] = grid[i - 1][0] + grid[i][0];
    }

    for (int i = 1; i < grid[0].length; i++) {
        grid[0][i] = grid[0][i - 1] + grid[0][i];
    }

    for (int i = 1; i < grid.length; i++) {
        for (int j = 1; j < grid[0].length; j++) {
            grid[i][j] = Math.min(grid[i - 1][j], grid[i][j - 1]) + grid[i][j];
        }
    }

    return grid[grid.length - 1][grid[0].length - 1];
}
