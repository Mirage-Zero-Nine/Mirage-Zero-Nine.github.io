Running Mongodb on Ubuntu 16.04

May 19, 2019

Make it works

This is a side problem I confronted when I tried to deploy a Node.js app with MongoDB on my private server. Although I follow the path of MongoDB Manual to install MongoDB, it just can not work, simply display the message when I check the status of MongoDB using systemctl status mongodb:

‚óè mongodb.service
   Loaded: not-found (Reason: No such file or directory)
   Active: inactive (dead)

The problem seems to be that the /lib/systemd/system/mongodb.service is missing. Therefore, the solution is to write a start script in systemd, in this way to make systemctl start mongodb work.

First, create file:

nano /lib/systemd/system/mongodb.service

I prefer nano but vim works in the same way.

Then paste following content into the file:

[Unit]
Description=MongoDB Database Service
Wants=network.target
After=network.target

[Service]
ExecStart=/usr/bin/mongod --config /etc/mongod.conf
ExecReload=/bin/kill -HUP $MAINPID
Restart=always
User=mongodb
Group=mongodb
StandardOutput=syslog
StandardError=syslog

[Install]
WantedBy=multi-user.target

Save and quit. Then use systemctl start mongodb can simply start it.

One more step, link the database path to MongoDB to start MongoDB via mongod.

sudo mkdir -p /data/db/

sudo chown `id -u` /data/db

Manipulate the service

Start MongoDB:

sudo systemctl start mongodb

Stop MongoDB:

sudo systemctl stop mongodb

Enable MongoDB on system start up:

sudo systemctl enable mongodb.service

Refresh the services:

sudo systemctl daemon-reload
