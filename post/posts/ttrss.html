Deploy Tiny Tiny RSS via Docker

April 5, 2019

Docker setup:

Clone this repo:

git clone https://github.com/HenryQW/docker-ttrss-plugins.git

Modify docker-compose.yml:

version: "3"
services:
  database.postgres:
    image: sameersbn/postgresql:latest
    container_name: postgres
    environment:
      - PG_PASSWORD=kBZ4V3Vz # please change the password
      - DB_EXTENSION=pg_trgm
    volumes:
      - ~/postgres/data/:/var/lib/postgresql/ # persist postgres data to ~/postgres/data/ on the host
    restart: always

  service.rss:
    image: wangqiru/ttrss:latest
    container_name: ttrss
    ports:
      - 26890:80		# change port here as well
    environment:
      - SELF_URL_PATH=http://142.93.70.153:26890/ # please change to your own domain
      - DB_HOST=database.postgres
      - DB_PORT=5432
      - DB_NAME=ttrss
      - DB_USER=postgres
      - DB_PASS=kBZ4V3Vz # please change the password
      - ENABLE_PLUGINS=auth_internal,fever # auth_internal is required. Plugins enabled here will be enabled for all users as system plugins
    stdin_open: true
    tty: true
    restart: always
    command: sh -c 'sh /wait-for database.postgres:5432 -- php /configure-db.php && exec s6-svscan /etc/s6/'

  service.mercury: # set Mercury Parser API endpoint to `service.mercury:3000` on TTRSS plugin setting page
    image: wangqiru/mercury-parser-api:latest
    container_name: mercury
    expose:
      - 3000		# do not change this
    restart: always

  service.opencc: # set OpenCC API endpoint to `service.opencc:3000` on TTRSS plugin setting page
    image: wangqiru/opencc-api-server:latest
    container_name: opencc
    environment:
      NODE_ENV: production
    expose:
      - 3000
    restart: always

Modify these items:

  • PG_PASSWORD
  • ports and SELF_URL_PATH, remember the port number should be same
  • DB_PASS, note that this should keep same as PG_PASSWORD

Run this command:

docker-compose up -d

The docker set up should be ready after all containers have been pulled to local place and running.

Set up Tiny Tiny RSS itself

Try to access the Tiny Tiny RSS via SELF_URL_PATH. If everything correct, then the explorer should display the login page. The default user name and password is admin and password. After first login it will remind you to change password.

After the password is set, modify some basic setting in preference page.

  • Default feed update interval - set to 1 hour
  • Enable API access - open it
  • Purge articles after this number of days - set to 30 days

Then save the configuration.

Next is the Mercury plugin and Fever plugin. Set Mercury Parser API endpoint to service.mercury:3000 and OpenCC API endpoint to service.opencc:3000. Fever plugin requires a password, create one and save it.

In Reeder, the server address is http://142.93.70.153:26890/plugins/fever/ and password is mentioned above. Email is admin.

Remember to keep a copy of previous subscription in OPML format.

Add Nginx And HTTPS Domain Support

Add Nginx and HTTPS domain to prevent direct access to your server port. This step is optional and requires a domain of your own.

Install Nginx

sudo apt-get update
sudo apt-get install nginx

Add an configuration file

nano /etc/nginx/conf.d/example.conf

A sample config file:

upstream ttrsstest {
  server 127.0.0.1:YOUR_TTRSS_PORT;
}

server {
    server_name  YOUR_DOMAIN;
    rewrite ^(.*)$ https://$host$1 permanent;
}

server {
    listen 443 ssl;
    gzip on;
    server_name  YOUR_DOMAIN;
		## use Certbot to add certification

    access_log /var/log/nginx/ttrssdev_access.log combined;
    error_log  /var/log/nginx/ttrssdev_error.log;

  ## add your domain below for regex
    if ($host ~* "^(.*?)\.example\.com$") {
                set $domain $1;
    }
		location / {
        proxy_redirect off;

        proxy_set_header  Host                $http_host;
        proxy_set_header  X-Real-IP           $remote_addr;
        proxy_set_header  X-Forwarded-Ssl     on;
        proxy_set_header  X-Forwarded-For     $proxy_add_x_forwarded_for;
        proxy_set_header  X-Forwarded-Proto   $scheme;
        proxy_set_header  X-Frame-Options     SAMEORIGIN;

        client_max_body_size        100m;
        client_body_buffer_size     128k;

        proxy_buffer_size           4k;
        proxy_buffers               4 32k;
        proxy_busy_buffers_size     64k;
        proxy_temp_file_write_size  64k;

				## assign your second level domain here if you have other purpose
        ## if ($domain ~* "^test$" ){
        ##    proxy_pass http://ttrsstest;
        ## }
        proxy_pass http://ttrssdev;
    }
}
