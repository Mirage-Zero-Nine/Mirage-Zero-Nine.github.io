Comparator in PriorityQueue

June 01, 2019

PriorityQueue is like a regular queue, but each element has a “priority” associated with it.

In PriorityQueue, it still follows FIFO rule when removing elements. However, the rule is “priority” instead of time. Hence, it uses a comparison function which imposes a total ordering of the elements as “priority”.

The elements of the priority queue are ordered according to their natural ordering, or by a Comparator provided at queue construction time, depending on which constructor is used. The natural ordering is same as Queue, and one “priority number” is assigned to the elements in PriorityQueue. The smaller that number is, the higher priority this element has. That is to say, this element will be removed earlier.

Sample:

class Cell implements Comparable<Cell> {
        int row, col, height;

        Cell(int row, int col, int height) {
            this.row = row;
            this.col = col;
            this.height = height;
        }

        public int compareTo(Cell o) {
            return this.height - o.height;      // sort by height, shorter cell has higher priority
        }
    }

Original problem solution: Trapping Rain Water II

In the sample code above, the compareTo is the comparator that assign a “priority” to queue. If the current cell’s height is shorter than given cell’s height, then it returns -1. Based on comparator, current cell has smaller value, and then has higher “priority”.

The comparator can be modify to any other order, as long as the comparator can be constructed.
